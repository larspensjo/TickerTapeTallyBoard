Gains Breakdown — Layout Mockups

Three layout concepts for replacing the two-line gains panel on the asset view. Same dark-theme tokens across all three. Toggle the scenario to see open vs. closed positions, gains vs. losses, populated dividends, and an Unavailable step.

Scenario

A — Waterfall

recommended

Running total from the base up. Each step floats from where the last ended; negative steps float down and render red. Most literal "how it builds up".

B — Rows + running subtotal

cheapest
StepAmountRunning

Reuses the existing list, adds a cumulative column. Precise and scannable; least visual but lowest effort — a good first increment.

C — Segmented bar

compact

One horizontal bar split into proportional segments. Compact and mobile-friendly, but labels are cramped — values move to a legend below.

Open identity: market value = cost basis + price + FX, total return = unrealized + realized + income. Closed identity: proceeds = cost basis(sold) + price + FX, total return = realized + income (no market value — shares no longer held). Dividends are mocked here as a future per-instrument field. An Unavailable step (e.g. missing FX) is hatched and excluded from the running total, distinct from a genuine 0.
